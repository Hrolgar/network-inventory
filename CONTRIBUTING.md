# Contributing

Thanks for considering contributing.

## Reporting Issues

Found a bug or have a feature request? Open an issue with:
- Description of the problem/feature
- Steps to reproduce (for bugs)
- Your environment (OS, Docker version, UniFi model, etc.)

## Pull Requests

1. Fork the repo
2. Create a feature branch (`git checkout -b feature/thing`)
3. Make your changes
4. Test it works (`docker-compose build && docker-compose up`)
5. Commit and push
6. Open a PR

Keep changes focused. One feature/fix per PR.

## Code Style

- Python: Follow PEP 8
- Keep it readable
- Comment non-obvious stuff

## Testing

Before submitting:
- Build the Docker image successfully
- Dashboard loads without errors
- Your changes actually work

## Questions

Open an issue if you're unsure about something.
